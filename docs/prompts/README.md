# Session Prompts

Rerunnable specifications for multi-session work across all yogananda projects. Each file is a complete, self-contained instruction set that could re-implement its scope from scratch.

**Usage:** Copy the Prompt section into a new Claude Code session targeting the specified repo. "What Already Exists" describes prerequisite state — "What to Build" is the work.

**Convention:** Files are named `stg-NNN-{slug}.md`. Each contains: target repo, model, governing FTRs, files to read, existing state, deliverables, model constraints.
